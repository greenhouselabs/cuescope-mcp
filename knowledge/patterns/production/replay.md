# Replay

Replay control is show-critical because it can affect recording, live playback, and operator workflow.

Common actions:

- Select channel A or B.
- Mark in/out.
- Play last event.
- Set replay speed.
- Return to live.

Guidance:

- Treat replay automation as Control Mode behavior.
- Review Mode should describe and plan replay actions, not execute them.
- Validate whether replay is enabled and whether channel context is known before suggesting operator actions.
- Avoid generating replay sequences that assume a specific event exists unless current state confirms it.
