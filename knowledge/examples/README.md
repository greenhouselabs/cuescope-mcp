# vMix Examples

These examples are review-first patterns for Review Mode. They are not instructions to execute live changes without operator review.

Example groups:

- `scripts/`: VB.NET review artifacts.
- `presets/`: preset notes that describe production structure, risks, and useful MCP questions.
- `xml-snapshots/`: before/after XML interpretation examples.
- `routing-diagrams/`: text routing diagrams for audio and show-flow reasoning.
- `troubleshooting/`: sanitized error/log fixtures with expected diagnosis shape and safe next checks.

Current preset examples include podcast, virtual-set interview, and paired-audio aux-bus video-show patterns.
Current troubleshooting examples include capture, audio, NDI, MCP startup, script, and API ambiguity cases.

Use examples when an MCP client needs a compact model of what "good output" should look like.
